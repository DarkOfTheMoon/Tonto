The choice of language

C++ seems the obvious choice since it is an object oriented language, and the goal of a program with simple self contained logical sub-units is most easily achieved with an object-oriented language. The unique features of an object-oriented language that facilitate logical design are the ability to define and use abstract data types in which the internal implementation and algorithms are hidden or encapsulated; and the ability to inherit or reuse old code in new contexts.

Fortran95 is not a true object oriented language, but it does have the facility tio define abstract data types. However, an inheritance mechanism is missing. The main advantages of Fortran95 over C++ for numerical work are:

  • It has a powerful inbuilt array handling syntax
  • It has an inbuilt complex data type
  • It has good aliasing properties for pointers, and good expression re-arrangement properties
  • It has a tradition of providing highly optimized compilers for heavy numerical computation
  • It is an internationally standardized parallel SIMD and MIMD programming language
  • It is fully backwards compatible with Fortran77, so legacy code can be used

The lack of any built-in inheritance mechanism in Fortran95 is also not so serious as it first looks. Provided a few simple coding styles are adhered to, inheritance can be implemented by hand by explicitly including old code. Minor modifications of the old code are required to deal with new types and routine calls. These modifications can be achieved with a single search-substitute command in an editor.
