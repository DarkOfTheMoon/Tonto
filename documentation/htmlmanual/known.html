Installing TONTO on a known system

The steps for installing TONTO on a known system are described below.

The example given is for someone running the Fujitsu compiler on Linux, and will install the source code in /usr/local/src/tonto, and the executable in /usr/local/bin.

If you encounter any problems, send us an e-mail: we would like to know. We cannot guarantee to help, but we will try. Some insight into problems that may occur are described in the section below.

  1. Decide on a place for the TONTO source code. This may be your home directory ~, or it may be a place where you usually place program sources, like /usr/local/src. A directory called "tonto" will be created in this place, which will contain the source code and documentation.

    • cd /usr/local/src

  2. If you do not have the source code, it can be downloaded from
    http://www.theochem.uwa.edu.au/tonto/.
    Copy the downloaded file tonto.tar.gz into the directory you have just changed to.

    • cp ~/tonto.tar.gz .

  3. Uncompress and untar the file tonto.tar.gz.
    (You will need the GNU free software program called gunzip)

    • gunzip -c tonto.tar.gz | tar xvf -

  4. Change to the "tonto" directory.

    • cd tonto

  5. There should now be a few files and directories, such as configure, Makefile.in, foofiles, and documentation. Run the "configure" script to set up the TONTO compilation to run on your machine. At this stage, you can tell it where you want the executables installed, via the --bindir= option to configure. See the configure section for more details or if there are any errors.

    • ./configure --bindir=/usr/local/bin

  6. Using an editor program, have a look at your site_config file and make sure it is correct. The name of this file is given to you whenever you run configure. If you change this file, be sure to run configure again, otherwise continue to the next step.

    • vi ./site_config/FUJITSU-LINUX 

      F90       = $(FFAST)
FFLAGS    := -Cpp -I. -Am -I./modules -M./new_modules -I./interfaces \
             -I./objects -I./f90 -f w -static-flib
FDEBUG    := -g -H aesu
FFAST     := -O -Komitfp -Kloop -Kauto -Kpreex -Keval -x-

  7. If you wish to use TONTO's debugging or memory checking facilities, then edit the macros file. This is described here. Most users should probably skip this step, as the default configuration does not use these.

  8. The configuration part is now complete.
    In the instructions below, if your make program is called gmake, substitute the word "make" for "gmake".

    To compile and install the main executable, type "make install".
    The main executable for TONTO, run_mol.x, will be compiled and installed into the /usr/local/bin directory (or the alternative directory if you ran configure with the --bindir= option).

    To build the documentation, type "make documentation".
    To see the documentation use your browser to open the file "documentation/index.html".
