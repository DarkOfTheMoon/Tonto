Problems installing TONTO

If you have read the following sections on possible problems and solutions and are still stuck, then feel free to ask us to help with your problems.
We cannot guarantee to help, but we will try.

There are a few problems you may experience in getting TONTO to work:

  • The configure script cannot find programs, or does not recognise your compiler.
  • The Fortran compiler complains while trying to compile TONTO.
Detailed information to help you with these separate issues is given in following sections. However, the first step is to decide which of the problems you have.

If the problem occurs while running the configure script, then see the configure documentation.

Problems with the compiler are indicated by messages which involve the Fortran compiler (say, f95 or ftn90), and may mention errors involving words or phrases like subroutine, incorrect syntax, or undeclared variable. Usually, a line of Fortran code will also be printed. See the compilers section.

Compiler specific issues

Compaq Fortran 5.3 for TRU64 and OSF/1
  • use build 1120 of the compiler or later, the original requires too much memory.
    You can download the patch for free from the Compaq web site.
 Digital Fortran 5.0 for OSF/1
  • run time libraries 376 or later must be used.
    Before this, the "for_cpusec" routine will be unresolved.
  • in the macros file, must have:
    • #define DEFAULT_NULL
     This is a compiler bug.
    The "configure" script should detect and fix this automatically.
Fujitsu Fortan for Linux
  • use version 1999/07/29 or later of the compiler, the original requires too much memory.
