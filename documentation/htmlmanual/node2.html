What is TONTO???

TONTO is a free computer package for doing quantum chemistry calculations.

The name TONTO was suggested by Anthony Russell, and means ``fool'' in Spanish.

The main objective of TONTO is to be simple--simple to understand, simple to use, and most importantly, simple to hack. However, because quantum chemical calculations are time consuming, efficiency is also an important issue; but efficiency is not more important than simplicity, for TONTO.

TONTO is intended for quantum chemists with new ideas, but who are extremely frustrated by the time it takes to understand and modify older, monolithic quantum chemistry packages to do what they want. TONTO is not for people who want to crank out the papers. If you want to do that, go and buy Gaussian or Spartan or HyperChem or ADF. TONTO is for the people who can't get these packages to do exactly what they want. TONTO is for the people who mainly write code.

TONTO is copyleft under the less restrictive GNU library licence.

If you want features added or corrected in TONTO, you can do it yourself and mail the new stuff to us! Your code contributions will be very much appreciated.

TONTO has been implemented in foo, an object oriented Fortran preprocessor which translates into Fortran95. Foo stands for object oriented Fortran in reverse. The main facility that foo allows is a dot notation to represent message passing between ``objects'' and ``methods'', as used in many object oriented languages. In foo, the dot stands for a routine call or a structure component separator, depending on the context. For example, 

   a.b   means   call b(a)
    .b   means   call b(self),  or   self%b
Foo also does a number of other minor things, like eliminating the need to use subroutine or function keywords, and the ability to use the end keyword to replace all control-flow terminators, like end do, end if, end subroutine, end function, etc ...

In the following parts of this document we go into the issues which affected the design of TONTO and foo, and also the specifications. The foo preprocessor is still under development, and preconditions and inheritance via code inclusion are planned.

If you wish to skip directly to view the code, the frame bar of the left will take you to the documentation for each of the available modules, which each encapsulates an abstract data type used in quantum chemical calculations. The documentation is generated directly from the foo code via the foo2html converter, and consists of a synopsis section and a list of routine specifications.
