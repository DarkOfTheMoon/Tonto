Critical requirements for encapsulation

Here is the list of critical requirements to write modules which encapsulate an abstract data type which are proposed for TONTO :

  • The first argument of each routine in a module is an object of the abstract type described by that module, and it is this object which is involved in manipulations by the routine.

  • The name of each module routine always begins with the name of the module followed by an underscore.

    STR_get_item(self) would be a routine which appears in module STR which described the abstract data type ``strings''. self is variable of type which represents ``strings''.

  • Every module routine appears in at least one generic interface, where the generic name is the same as the routine name without the preceding module name.

    The following interface must appear for routine STR_get_item: 
    interface _get_item
   module procedure STR_get_item
end interface

  • If the abstract data type encapsulated in the module is realized by a Fortran95 derived type, then the type definition is placed in a common module TYPES.

    The TYPES module is the only module which does not encapsulate an abstract data type.
