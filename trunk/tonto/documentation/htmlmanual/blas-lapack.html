Compiling the BLAS and LAPACK libraries

If you do not have BLAS and LAPACK, you will have to compile these. Note that if your vendor has their own version, you should use that as it is probably much faster.

Tonto is distributed with the subset of BLAS and LAPACK that it needs. These files are in the lapack directory. To compile BLAS and LAPACK, simply do : 
   make blas
   make lapack
in the top directory. This will create the appropriate libraries and put them into the directory where the other TONTO objects will be compiled and placed.

Depending on your compiler, you may have to add -L$(objdir) to the LIBS line in your site_config file. Otherwise, this line will probably just look like 
   LIBS = -llapack -lblas

If you have BLAS and LAPACK which were compiled using a different compiler, you may need to specify additional libraries to use these. For example, on a Redhat Linux system with pre-installed BLAS and LAPACK, use something like 
LIBS    := -L usr/lib/gcc-lib/i386-redhat-linux/2.96 -lg2c -llapack -lblas
This will vary depending on your operating system / distribution.

See the section called “The LIBS variable” for more information.
